# CyanPrint Architecture

CyanPrint is a comprehensive templating platform designed to be language-agnostic and extensible. It consists of several components that work together to provide a seamless experience for creating projects from templates.

## System Components

CyanPrint's architecture is divided into several key components:

1. **CyanPrint CLI (Iridium)** - The command-line interface that users interact with to create projects from templates.

2. **CyanPrint Registry (Zinc + Argon)** - A central repository for templates where developers can publish and share their templates.
   - **Zinc** - The backend service that stores and manages templates
   - **Argon** - The frontend web interface for discovering and browsing templates

3. **CyanPrint Coordinator (Boron)** - Manages the execution of templates and plugins.

4. **Template Development SDKs (Helium)** - Libraries for creating CyanPrint templates in various languages (.NET, TypeScript, JavaScript, Python).

## Workflow

1. Users interact with the CyanPrint CLI (Iridium) to search for templates and create projects.
2. Templates are fetched from the Registry (Zinc backend, browsed via Argon frontend).
3. The Coordinator (Boron) executes the template, running any associated plugins and processors.
4. Template developers use the SDKs (Helium) to create and test templates before publishing them to the Registry.

## Key Features

- **Language Agnostic**: Supports templates in any programming language
- **Extensible**: Custom plugins and processors can be created to extend functionality
- **Central Registry**: Easy discovery and sharing of templates
- **Rich Templating**: Advanced features beyond simple text substitution
